# 09 Korean Stock Spec

## 1. 문서 목적

- 한국주식을 어떤 범위까지 지원하고, 어떤 데이터 구조와 전략군으로 시작할지 정의한다.
- 이 문서는 구현 지시용 PRD이며, 대화 과정은 남기지 않고 `합의된 정책`만 기록한다.

## 2. 한 줄 정의

- 한국주식 지원은 `백테스트 + 페이퍼트레이딩 중심의 보수적 확장 영역`이며, 대형주와 중저빈도 전략부터 시작해 장기적으로 멀티브로커 구조로 확장한다.

## 3. 범위

### 포함

- 국내 증권 API 기반 시세 데이터 연동
- broker adapter 구조
- 백테스트
- 페이퍼트레이딩
- 대형주 중심 종목군
- 추세추종 + 브레이크아웃 전략군
- 대시보드 노출
- 오케스트레이터 입력

### 제외

- 실전 자동매매
- 소형주/초저유동성 종목 적극 운용
- 고빈도 초단타 전략
- 다중 브로커 동시 운영

## 4. 핵심 원칙

- 한국주식은 암호화폐와 다른 시장으로 취급한다.
- 장 운영 시간, 휴장일, 장 시작/마감 이벤트를 명시적으로 반영한다.
- 초기에는 종목군과 전략군을 좁게 시작한다.
- 브로커는 1개부터 구현하되, 구조는 adapter 기반으로 만든다.
- 실전 매매는 현재 범위에서 제외하고, 백테스트와 페이퍼 검증에 집중한다.

## 5. 브로커 구조

### 5.1 기본 정책

- `broker adapter` 구조를 전제로 설계한다.
- 첫 구현은 `1개 브로커`부터 시작한다.

### 5.2 의미

- 데이터 모델과 실행 모델을 브로커별로 분리할 수 있다.
- 장기적으로 다른 증권사 API 추가가 쉬워진다.
- 멀티유저 플랫폼 확장 시 특정 브로커 종속을 완화할 수 있다.

## 6. 데이터 범위

### 6.1 기본 데이터

- OHLCV
- 거래대금
- 체결 데이터
- 호가 데이터
- 거래정지/관리종목/이벤트 상태

### 6.2 파생 지표

- EMA
- RSI
- MACD
- ATR
- Bollinger
- volatility score
- liquidity score
- sector strength proxy

### 6.3 시장 시간대 데이터

- 장 시작
- 장 마감
- 휴장일
- 반장/특수 일정

## 7. 종목 유니버스 정책

### 7.1 기본 유니버스

- 대형주 중심
- 거래대금 충분한 종목 중심
- 데이터 안정성이 높은 종목 중심

### 7.2 제외 대상

- 초저유동성 종목
- 급격한 이벤트성 왜곡이 빈번한 종목
- 데이터 신뢰도가 낮은 종목

### 7.3 바스켓 예시

- KOSPI 대형주 바스켓
- KOSDAQ 대형/중형 중 거래대금 상위 바스켓

## 8. 1차 전략군

### 8.1 추세추종

- EMA 기반 추세 전략
- 이동평균 돌파형
- 추세 필터 + 눌림 진입 보조 가능

### 8.2 브레이크아웃

- 박스권 상단 돌파
- Donchian/채널 돌파
- 거래대금 증가 동반 브레이크아웃

### 8.3 후순위 전략

- 평균회귀
- 모멘텀 로테이션
- 장 초반 이벤트 전략
- 고빈도 체결/호가 전략

## 9. 시간대 특화 규칙

한국주식은 24시간 시장이 아니므로 시간대 규칙이 중요하다.

### 9.1 기본 원칙

- 장중과 장외를 명확히 구분한다.
- 장 시작과 장 마감은 별도 이벤트 구간으로 취급한다.
- 휴장일과 특수 일정은 캘린더 기반으로 반영한다.

### 9.2 장 시작 구간

- 변동성과 거래대금이 급격히 커질 수 있다.
- 브레이크아웃 전략에는 기회가 될 수 있지만, 슬리피지와 왜곡도 크다.

### 9.3 장 마감 구간

- 리밸런싱과 종가 왜곡 가능성이 있다.
- 종가 기반 전략은 별도 검증 필요

## 10. 연구 루프 적용 방식

### 10.1 초기 범위

- 대형주 중심
- 중저빈도 전략 중심
- 추세추종 + 브레이크아웃 우선

### 10.2 파라미터 탐색 방향

- EMA 기간
- 돌파 lookback
- 거래대금 필터
- volatility filter
- holding period

### 10.3 검증 원칙

- 거래비용 반영
- 장 시간대 반영
- 휴장일 반영
- 종목군별 데이터 결측 확인

## 11. 페이퍼트레이딩 적용 방식

### 11.1 목적

- 한국주식 전략의 실시간 검증
- 백테스트와 실제 장중 움직임 차이 확인
- 장 시작/마감 체결 품질 검증

### 11.2 기본 자본

- [05_PAPER_TRADING_SPEC.md](/root/work/coin-autopilot/PRD/05_PAPER_TRADING_SPEC.md)의 한국주식 기본 세션 자본을 따른다.
- 현재 기준: `10,000,000 KRW`

### 11.3 체결 모델

- 대형주 중심이므로 암호화폐 알트보다 보수성이 덜해도 되지만, 장 시작/마감 이벤트 가산은 반드시 반영한다.

## 12. 오케스트레이터 연계

- 한국주식은 별도 `asset_slot` 또는 `basket_slot`으로 관리한다.
- 암호화폐와 동일한 점수 체계를 쓰되, 시장 시간대와 장중 이벤트를 추가 반영한다.
- 실전 배치가 아니라 페이퍼 중심 후보군으로 우선 취급한다.

## 13. 대시보드 반영 항목

- 한국주식 노출 비중
- 현재 활성 전략
- 현재 포지션
- 거래대금/유동성 상태
- 장 시작/마감 이벤트 상태
- 연구 후보 상태

## 14. 리스크 관리 반영 항목

- 단일 종목 최대 노출
- 장 시작/마감 슬리피지 가산
- 저유동성 종목 제외
- 장외/휴장일 신규 진입 금지

## 15. 성공 기준

- broker adapter 구조로 데이터 수집이 가능하다.
- 대형주 중심 유니버스에서 백테스트와 페이퍼트레이딩이 가능하다.
- 추세추종 + 브레이크아웃 전략군이 연구 루프에 편입된다.
- 오케스트레이터와 대시보드에서 한국주식 슬롯을 동일 방식으로 다룰 수 있다.

## 16. 오픈 질문

- 첫 브로커를 어느 증권사로 확정할지
- 한국주식 1차 유니버스를 KOSPI 대형주로만 제한할지
- 섹터 강도 프록시를 어떤 방식으로 계산할지
- 장 시작 구간을 몇 분까지 별도 이벤트 구간으로 볼지
