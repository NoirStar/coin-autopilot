# 06 Risk Management Spec

## 1. 문서 목적

- 시스템 전반의 손실 제한, 노출 제한, 레버리지 통제, 자동 중지 규칙을 정의한다.
- 이 문서는 구현 지시용 PRD이며, 대화 과정은 남기지 않고 `합의된 정책`만 기록한다.

## 2. 한 줄 정의

- 리스크 관리 계층은 전략보다 상위에서 동작하며, 손실 확대, 시장 이상, 데이터 이상, 실행 실패가 발생할 때 시스템의 노출을 줄이거나 강제로 멈추는 안전장치다.

## 3. 범위

### 포함

- 일 손실 제한
- 세션 손실 제한
- 전략별 MDD 제한
- 자산군별 노출 한도
- 단일 자산 노출 한도
- 레버리지 제한
- 서킷 브레이커
- 데이터/실행 이상 감지
- 자동 중지 및 승인 요구 규칙

### 제외

- 전략 내부 손절 로직 상세
- 주문 라우팅 상세
- 브로커/거래소 API 명세

## 4. 핵심 원칙

- 수익보다 생존이 우선이다.
- 모든 전략은 리스크 관리 계층 아래에서만 동작한다.
- 손실 제한은 전략별, 세션별, 자산군별로 중첩 적용한다.
- 데이터 이상과 실행 이상도 가격 손실만큼 위험하게 취급한다.
- 페이퍼에서는 자동 중지가 가능해야 하며, 실전은 더 보수적으로 제한한다.

## 5. 리스크 계층 구조

### 5.1 전략 수준

- 개별 전략 손절
- 전략별 최대 MDD
- 전략별 허용 포지션 크기

### 5.2 세션 수준

- 페이퍼 세션 일 손실 한도
- 세션 누적 손실 한도
- 반복 실패 한도

### 5.3 자산 수준

- 단일 자산 최대 노출
- 단일 자산 최대 동시 전략 수
- 이벤트 발생 시 자산 단위 중지

### 5.4 자산군 수준

- 암호화폐 전체 노출 한도
- 한국주식 전체 노출 한도
- 고위험 전략군 전체 노출 한도

### 5.5 시스템 수준

- 전체 시스템 일 손실 한도
- 시스템 장애 시 신규 진입 차단
- 서킷 브레이커

## 6. V2 기본 리스크 정책

### 6.1 단일 자산 최대 노출

- 기본값: 세션 자본의 `25%`

의미:

- 하나의 자산이 전체 세션을 좌우하지 않도록 제한한다.
- 바스켓 전략은 개별 자산 비중 합산 시에도 이 원칙을 고려한다.

### 6.2 자산군 최대 노출

- 암호화폐 전체: 세션 자본의 `60%`
- 한국주식 전체: 세션 자본의 `60%`
- 고위험 전략군 전체: 세션 자본의 `20%`

의미:

- 자산군 내 집중을 막는다.
- 고위험 이벤트 전략은 제한된 예산에서만 작동하도록 한다.

### 6.3 일 손실 한도

- 기본값: 세션 자본 대비 `3%`

정책:

- 일 손실 한도 도달 시 신규 진입 중지
- 기존 포지션은 전략/리스크 정책에 따라 축소 또는 종료
- 당일 세션은 `paused` 상태로 전환 가능

### 6.4 세션 누적 손실 한도

- 기본값: 세션 자본 대비 `10%`

정책:

- 누적 손실 한도 도달 시 세션 중지
- 자동 재개 금지
- 사용자 검토 후 재개

### 6.5 전략별 허용 MDD

- 저위험 전략: `10% 이하`
- 중위험 전략: `15% 이하`
- 고위험 전략: `20% 이하`

정책:

- 전략 MDD가 기준 초과 시 감점
- 지속 초과 시 `paper_paused`
- 심각 초과 시 `retired` 후보

## 7. 레버리지 정책

### 7.1 V2 기본값

- 기본 연구/페이퍼 레버리지 범위:
  - `1x ~ 5x`
- 상한 실험:
  - `10x`

### 7.2 기본 제한

- 5x 초과는 기본 배포 전략으로 사용하지 않는다.
- 10x는 별도 고위험 실험군으로만 허용한다.
- 고변동 구간에서는 허용 레버리지를 자동 하향할 수 있다.

### 7.3 자동 제한 조건

- 변동성 급등
- 청산 이벤트 급증
- 펀딩 급변
- execution quality 악화

## 8. 손실 및 중지 규칙

### 8.1 신규 진입 중지 조건

- 일 손실 한도 초과
- 데이터 결측률 급증
- 주문 실패 반복
- warning/caution 심화
- 유동성 급감
- 슬리피지 급등

### 8.2 비중 축소 조건

- MDD 한도 근접
- 체결 품질 악화
- 변동성 급등
- 특정 자산 집중도 상승

### 8.3 세션 중지 조건

- 세션 누적 손실 한도 초과
- 반복 주문 실패
- 데이터 이상 지속
- AI 리뷰 결과 강한 중지 권고
- 오케스트레이터가 리스크 이벤트로 판단

## 9. 서킷 브레이커

### 9.1 목적

- 블랙스완 또는 시스템 이상 시 자동으로 신규 진입을 막고 노출을 줄인다.

### 9.2 발동 조건 예시

- 단시간 급변동
- 청산 이벤트 급증
- 거래소/브로커 연결 이상
- 주문 실패율 급등
- 호가/체결 데이터 이상
- 김치 프리미엄 급변
- warning/caution 급증

### 9.3 발동 시 동작

- 신규 진입 차단
- 미체결 주문 취소
- 필요 시 포지션 축소
- 세션 또는 자산군 `paused`
- 대시보드/텔레그램/디스코드 경고

## 10. 데이터 및 실행 리스크

### 10.1 데이터 리스크

- 캔들 결측
- 틱 누락
- 호가 공백
- 시간 정렬 오류
- 김프 계산 실패

### 10.2 실행 리스크

- 부분체결 반복
- 주문 실패율 증가
- 지연시간 급등
- 호가 깊이 부족
- 슬리피지 모델과 실제 편차 확대

### 10.3 대응 원칙

- 리스크는 전략 성과 감점에 반영한다.
- 반복되면 세션 중지 또는 후보 제외로 이어진다.

## 11. 경고 등급

### 11.1 Info

- 단순 참고 수준
- 예: 경미한 변동성 확대

### 11.2 Warning

- 운영자가 확인해야 하는 수준
- 예: MDD 한도 근접, 부분체결 빈도 증가

### 11.3 Critical

- 자동 중지 또는 승인 필요 수준
- 예: 주문 실패 반복, 일 손실 한도 초과, 데이터 이상 심화

## 12. 오케스트레이터 연계

- 리스크 관리 계층은 오케스트레이터보다 우선한다.
- 리스크 이벤트가 발생하면 전략 점수보다 먼저 중지/축소를 판단한다.
- 오케스트레이터는 리스크 상태를 점수 산정에 반영한다.
- 고위험 전략은 동일 성과라도 기본 감점이 적용될 수 있다.

## 13. 페이퍼트레이딩 연계

- 페이퍼 세션은 리스크 한도에 도달하면 자동으로 `paused` 또는 `stopped` 될 수 있다.
- 페이퍼에서 반복적으로 리스크 한도를 넘는 전략은 실전 후보로 승격하지 않는다.
- 리스크 초과 이력은 연구 루프의 전략 평가에도 반영된다.

## 14. AI 연계

AI는 리스크 관리에서 `설명과 재조정 제안` 역할을 한다.

### AI가 할 수 있는 것

- 손실 확대 원인 분석
- 변동성 급등 원인 요약
- 파라미터 하향 제안
- 비중 축소 제안
- 전략 중지 권고

### AI가 하지 않는 것

- 리스크 한도 무시
- 손실 한도 초과 상태에서 자동 재진입 강행
- 검증 없는 레버리지 확대

## 15. 대시보드 노출 항목

- 일 손실 현황
- 세션 누적 손실 현황
- 자산별 노출 비중
- 자산군별 노출 비중
- 전략별 MDD 상태
- 레버리지 상태
- 현재 경고 등급
- 서킷 브레이커 발동 여부

## 16. 저장해야 할 리스크 데이터

- loss limit breach history
- drawdown snapshots
- exposure snapshots
- leverage change history
- pause/stop reason
- circuit breaker event
- warning level history
- ai risk review summary

## 17. V2 성공 기준

- 전략보다 상위에서 리스크 중지 규칙이 적용된다.
- 일 손실, 누적 손실, 노출 한도가 자동으로 계산된다.
- 리스크 이벤트가 발생하면 오케스트레이터와 페이퍼 세션이 즉시 반응한다.
- 리스크 초과 전략은 자동 감점 또는 중지된다.
- 대시보드에서 현재 리스크 상태를 한눈에 볼 수 있다.

## 18. 오픈 질문

- 일 손실 한도를 3%보다 더 보수적으로 둘지
- 단일 자산 최대 노출 25%를 더 낮출지
- 고위험 전략군 전체 노출 20%를 더 낮출지
- 한국주식과 암호화폐의 리스크 한도를 동일하게 둘지
