# 07 Dashboard Spec

## 1. 문서 목적

- 사용자가 시스템 상태를 가장 빠르게 이해할 수 있도록 대시보드 정보구조와 핵심 화면 구성을 정의한다.
- 이 문서는 구현 지시용 PRD이며, 대화 과정은 남기지 않고 `합의된 정책`만 기록한다.

## 2. 한 줄 정의

- 대시보드는 BTC 상태판이 아니라, 현재 운용 상태, 자산별 전략 배치, 시장 상황, 연구 진행 상황, 리스크 경고를 한눈에 보여주는 `트레이딩 대시보드`이다.

## 3. 핵심 원칙

- 첫 화면은 `트레이딩 대시보드`이어야 한다.
- 특정 자산보다 `현재 운용 상태`가 먼저 보여야 한다.
- 단일 최고 전략보다 `자산별 전략 배치 상태`가 중요하다.
- 성과만이 아니라 리스크, 승인 대기, 연구 진행 상황이 함께 보여야 한다.
- 초보 사용자도 지금 무엇이 일어나고 있고 무엇을 해야 하는지 바로 이해할 수 있어야 한다.

## 4. 대시보드의 역할

대시보드는 아래 질문에 즉시 답해야 한다.

- 지금 돈은 어디에 어떻게 배치돼 있는가
- 현재 포지션과 손익은 어떤가
- 어떤 자산에 어떤 전략이 적용 중인가
- 왜 그런 전략이 적용됐는가
- 시장이 지금 어떤 상태인가
- 연구 루프는 지금 무엇을 하고 있는가
- 지금 당장 내가 확인하거나 승인해야 할 것이 있는가

## 5. 기본 화면 구조

### 5.1 트레이딩 대시보드

기본 첫 화면은 `트레이딩 대시보드` 하나로 정의한다.

핵심 구역:

1. 상단 상태 바
2. 운용 요약 카드
3. 오케스트레이션 보드
4. 포지션/세션 패널
5. 시장 상황 패널
6. 연구 루프/백테스트 현황
7. 리스크/승인/AI 액션 센터

## 6. 정보 우선순위

### 6.1 1순위: 현재 운용 상태

사용자가 첫 3초 안에 봐야 하는 정보:

- 총 가상 자산
- 오늘 손익
- 누적 성과
- 활성 포지션 수
- 페이퍼 실행 세션 수
- 승인 대기 항목 수

### 6.2 2순위: 자산별 전략 배치

- 어떤 자산에 어떤 전략이 붙어 있는지
- 현재 운용 모드가 무엇인지
- 유지/교체 판단 이유가 무엇인지

### 6.3 3순위: 현재 시장 상태

- 암호화폐 전반 상태
- 한국주식 전반 상태
- 주요 필터 신호
- 리스크 이벤트

### 6.4 4순위: 연구 및 검증 상태

- 현재 돌아가는 연구 작업
- 최근 완료된 상위 후보
- 페이퍼 후보 승격 대기

### 6.5 5순위: 액션 필요 항목

- 승인 요청
- 성과 악화 경고
- 리스크 경고
- AI 재분석 결과

## 7. 트레이딩 대시보드 상세 구성

### 7.1 상단 상태 바

노출 항목:

- 총 가상 자산
- 오늘 손익
- 누적 수익률
- 활성 전략 수
- 실행 중 페이퍼 세션 수
- 승인 대기 건수
- 현재 경고 등급

목적:

- 전체 상태를 한 줄에서 빠르게 파악

### 7.2 운용 요약 카드

노출 항목:

- 암호화폐 노출 비중
- 한국주식 노출 비중
- 고위험 전략 비중
- 현재 일 손실 진행률
- 현재 최대 드로우다운 진행 상태

목적:

- 수익과 리스크를 동시에 요약

### 7.3 오케스트레이션 보드

이 대시보드의 핵심 구역이다.

각 카드에 포함할 항목:

- 자산명
- 현재 전략
- 운용 모드
- 현재 상태
- 전략 점수
- 최근 변경 시점
- 유지/교체 이유 요약
- AI 개입 여부
- 승인 대기 여부

예시:

- `BTC-OKX-SWAP -> EMA Trend`
- `KRW-XRP-UPBIT -> Liquidity Trend`
- `KRX-005930 -> Breakout`

목적:

- "무엇이 어디에 왜 배치됐는가"를 한눈에 보여준다.

### 7.4 포지션/세션 패널

분리 원칙:

- 실전 포지션과 페이퍼 포지션은 시각적으로 분리한다.
- 초기에는 페이퍼가 중심이므로 `paper_running` 세션이 우선 노출된다.

노출 항목:

- 자산
- 방향
- 진입 전략
- 현재 손익
- 보유 시간
- 현재 리스크 상태
- 부분체결 여부

### 7.5 시장 상황 패널

암호화폐:

- 변동성 상태
- 펀딩비
- OI
- 롱/숏 비율
- 김치 프리미엄
- warning/caution 요약

한국주식:

- 시장 추세
- 거래대금
- 섹터 강도
- 장 시작/마감 이벤트 상태

목적:

- 오케스트레이터 판단 배경을 사용자가 이해할 수 있게 한다.

### 7.6 연구 루프 / 백테스트 현황

노출 항목:

- 실행 중 작업 수
- 큐 대기 작업 수
- 최근 완료 작업
- 상위 후보 전략
- `paper_candidate` 전략
- 실패/재시도 작업

목적:

- 지금 시스템이 무엇을 탐색하고 있는지 투명하게 보여준다.

### 7.7 리스크 / 승인 / AI 액션 센터

노출 항목:

- 일 손실 경고
- MDD 한도 근접
- 승인 필요 세션
- 승인 필요 전략 교체
- AI 재분석 결과
- 자동 중지된 세션

목적:

- 사용자가 "지금 무엇을 해야 하는지" 바로 알 수 있게 한다.

## 8. 화면 진입 구조

### 8.1 홈

- 트레이딩 대시보드

### 8.2 세부 진입

- 전략 상세
- 자산 상세
- 페이퍼 세션 상세
- 연구 결과 상세
- 리스크 상세
- 승인 상세

원칙:

- 홈에서 핵심 상태를 보고, 필요 시 세부 화면으로 이동한다.

## 9. 카드/리스트 우선 표현 방식

### 9.1 카드형이 적합한 영역

- 상단 상태 바
- 운용 요약
- 오케스트레이션 보드
- AI 액션 요약

### 9.2 테이블형이 적합한 영역

- 연구 루프 결과
- 백테스트 상위 후보
- 승인 대기 항목
- 세션 로그

### 9.3 색상/상태 표현 원칙

- 수익과 손실은 일관된 색 체계 사용
- 경고 등급은 `info / warning / critical`
- 승인 대기는 별도 강조 색상 사용
- 자동 중지 상태는 명확한 비활성 처리

## 10. 초보 사용자 친화 규칙

- 전문 용어는 바로 보조 설명을 제공한다.
- 점수와 상태에는 이유 요약이 함께 보여야 한다.
- "좋다/나쁘다"보다 "왜 그런 판단이 나왔는지"를 보여준다.
- 승인 요청은 숫자만 보여주지 말고 변경 이유를 같이 보여준다.

## 15. 알림 연계

- 대시보드에서 본 액션 항목은 텔레그램/디스코드 알림과 연결된다.
- 단, 모든 알림을 홈에 똑같이 쌓지 않고 `중요도` 기준으로 필터링한다.

## 16. 성공 기준

- 첫 화면에서 현재 운용 상태를 바로 이해할 수 있다.
- 자산별 전략 배치 상태를 한눈에 볼 수 있다.
- 연구 루프와 페이퍼트레이딩 진행 상황을 홈에서 확인할 수 있다.
- 승인 대기와 리스크 경고를 놓치지 않게 볼 수 있다.
- 초보 사용자도 다음 액션을 이해할 수 있다.

## 17. 오픈 질문

- 홈에서 포지션 패널과 연구 루프 패널 중 어느 쪽을 더 크게 줄지
- 암호화폐와 한국주식을 같은 보드에서 보여줄지, 탭으로 나눌지
- AI 액션 센터를 별도 영역으로 둘지, 승인/리스크와 합칠지
